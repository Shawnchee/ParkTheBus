## 1. Summary

ParkTheBus is a peer-to-peer football prediction market. A bettor posts a Request-For-Quote (RFQ) with
stake, minimum odds, and expiry, locking a 0.01 SOL refundable spam deposit. Market makers submit free
competing quotes; accepting one atomically escrows the bettor's stake and pulls the MM's collateral
(`collateral = stake × (odds − 1)`) into a per-position PDA. A native on-chain council settles each
market by threshold vote (`BettorWins` / `MmWins` / `Void`); once the threshold is met, anyone can crank
`execute_settlement` to release the escrow and update on-chain reputation. **Value custodied:** native
SOL only (no SPL tokens). **Core trust assumption:** the threshold-of-N council is the honest, available
sole arbiter of every outcome — and, until set otherwise, the upgrade-authority holder is honest.

## 5. Token Model (if applicable)

- Token program(s) supported: **none** — value is held as native SOL lamports inside program-owned PDAs.
  Branch 8 (SPL / Token-2022) is **N/A**.
- Lamport movement uses direct mutation (`util.rs:26-38 move_lamports`) for program-owned debits, and a
  System-Program CPI transfer (`anchor_lang::system_program::transfer`) for user-signed deposits.
- *Future:* the roadmap notes USDC settlement — at that point branch 8 becomes mandatory (ATA
  validation, decimals, fee-on-transfer, Token-2022 extensions).

## 6. Economic Invariants

1. **Escrow conservation** — `payout_amount = stake + collateral`; disbursed ≤ pot. Enforced at
   `execute_settlement.rs:90-93` + `move_lamports` calls (`:118-132`).
2. **Non-negative vault** — `mm_vault` never drops below rent-exempt after `accept_quote`. Enforced at
   `util.rs:42-45` + `accept_quote.rs:86-89`.
3. **Collateral > 0 (MM always at risk)** — odds > 1.0x ⇒ collateral ≥ 1. Enforced at
   `submit_quote.rs:57-59`, `accept_quote.rs:81-82`.
4. **Single disbursement** — settlement executes once. Enforced at `execute_settlement.rs:74,143` + Anchor
   `close`.
5. **Deposit refunded exactly once** — `rfq.deposit` zeroed after refund (`execute_settlement.rs:135-139`).
6. **Overflow-safe value math** — `overflow-checks = true` (`Cargo.toml:8`) plus `checked_*`/`u128`
   intermediates in `util.rs:6-21`.

## 7. Governance

- **Upgrade authority (launch → target):** currently undocumented (**F-02, HIGH, Launch**). Target: Squads
  Protocol v4 M-of-N multisig + timelock before mainnet, or `--final` immutable with a pause/migration
  story first. Must be published.
- **Admin powers (bounded?):** `config.authority` is recorded at `initialize.rs:43` but **never read** —
  today the on-chain admin has *zero* blast radius (it cannot move funds, change council, or pause). The
  gap is the inverse: there is **no** council/threshold rotation or key-recovery path (**F-01, HIGH,
  Launch**), so a lost council key permanently locks escrow. Add `update_council` gated by
  `has_one = authority` + `authority: Signer`, re-validating size/threshold/no-dupes.
- **Settlement authority:** native on-chain threshold-of-N council vote (`size/2 + 1` strict majority);
  party-exclusion and no-double-vote enforced. No single admin can settle. Council is a fully trusted
  arbiter by design (**F-INFO-1**).
- **Emergency pause:** none. Posture is "fail-closed" (funds stay escrowed if consensus isn't reached),
  which is acceptable for an MVP but means there is no circuit breaker for an incident — revisit before
  mainnet alongside the upgrade-authority decision.

## 8. Open Design Questions

1. **Upgrade-authority policy (F-02)** — multisig+timelock vs `--final`? Blocks mainnet.
2. **Council rotation / recovery (F-01)** — add `update_council`? Required to avoid permanent fund lock on
   a lost key.
3. **Parlay leg finality (F-03/F-05)** — make leg results one-way and/or freeze them once a settlement
   vote exists, to close the single-member griefing path.
4. **Quote improvability (F-04)** — keep `init` (and fix the `state.rs` comment) or support `update_quote`?
5. **Pause / circuit breaker** — add a guardian pause that never blocks user exit, before mainnet?
