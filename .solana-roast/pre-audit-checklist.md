# Pre-Audit Checklist — ParkTheBus

> Generated by `solana-roast`. Hand this to an external auditor or the kit's audit/scanner skills — it
> tells them what was already designed-for, so they can focus on confirming it in code. Each item:
> `[x]` enforced, `[ ]` open, `[-]` N/A. Cites `file:line` where enforced.
>
> Design-stage scores: **Code Safety 7/10 · Launch Readiness 6/10** (0 CRITICAL / 2 HIGH / 2 MEDIUM / 2 LOW).
> Code-Safety items below are largely green; the two open HIGHs both sit under **Upgrade & Governance**
> (Launch Readiness).

## Accounts & PDAs
- [x] Every data account read via typed `Account`/`InterfaceAccount` (owner + discriminator checked) — all `#[derive(Accounts)]` structs use `Account<'info, T>`; `UncheckedAccount` only for pinned recipients (`accept_quote.rs:25-27`, `execute_settlement.rs:40-46`)
- [x] All PDAs use canonical bump, stored at init, reused (no user-supplied bump) — `state.rs` `bump` fields; reused via `bump = x.bump` (e.g. `submit_quote.rs:13`, `accept_quote.rs:13`)
- [x] PDA seeds uniquely identify role; no cross-domain PDA sharing — `constants.rs:21-35` (distinct seeds per role)
- [x] Account relationships enforced (`has_one` / key checks) — `has_one = bettor`/`market_maker` (`accept_quote.rs:14`, `execute_settlement.rs:27-28`); `#[account(address = quote.market_maker)]` (`accept_quote.rs:26`)
- [x] Duplicate-mutable-account cases rejected — Anchor 1.0 rejects duplicate mutable accounts by default; no instruction passes the same writable PDA twice
- [~] `init` used (or `init_if_needed` proven re-init-safe) — `init` for one-shot accounts; `init_if_needed` on `MmVault`/`SettlementAccount`/`ReputationAccount` re-writes identity fields each call (idempotent) **but** `submit_quote` uses `init`, contradicting the "improvable quote" comment (**F-04, MEDIUM, Code**) — `submit_quote.rs:16`, `state.rs:128`

## Authority & Signers
- [x] Every privileged account is a `Signer` / `is_signer` verified (no key-only checks) — every handler takes a `Signer` (e.g. `sign_settlement.rs:38`, `execute_settlement.rs:67`)
- [x] Each signer bound to its resource (`has_one` / `require_keys_eq!`) — `has_one` on rfq/position; `config.is_council(member)` for council ixs (`approve_market.rs:26`, `sign_settlement.rs:49`)
- [x] Admin vs. user privilege separated — council vs bettor vs MM vs permissionless cranker all distinct
- [ ] Authority transfer is two-step / rejects default pubkey — **no authority/council transfer exists at all** (**F-01, HIGH, Launch**) — `lib.rs:20-85`, `state.rs:71-80`
- [x] Complete instruction→authority access matrix exists (design-spec §4)

## CPI & Composability
- [x] Every CPI target program ID is pinned/typed — only CPI is `system_program::transfer` via `Program<'info, System>` (`post_rfq.rs:38`, `deposit_collateral.rs:22`, `accept_quote.rs:48`); no untrusted external CPI
- [x] PDA signing scope is minimal / resource-scoped — program-owned debits use direct lamport mutation (`util.rs:26-38`), no `invoke_signed` with broad seeds
- [x] Account & signer/writable propagation reviewed — user deposits signed by the user; program debits only from program-owned PDAs
- [x] State mutated before external CPIs (checks-effects-interactions) — no external/untrusted CPI; settlement state set before/around internal lamport moves (`execute_settlement.rs:118-143`)
- [-] Token-2022 transfer hooks treated as reentrancy points — N/A (no tokens)

## State & Data
- [x] Type discrimination enforced (no manual deserialize of program data) — all via typed `Account<_>`
- [x] Re-initialization impossible / guarded — one-shot `init` accounts; `init_if_needed` accounts guard state by status checks (e.g. `settlement.executed`, `s.vote_count == 0` branch in `sign_settlement.rs:74`)
- [x] Account close zeroes data + invalidates discriminator (revival-safe) — Anchor `close = bettor` on rfq/position (`cancel_rfq.rs:12`, `expire_rfq.rs:12`, `execute_settlement.rs:24`)
- [-] `realloc` zeroes new bytes, correct payer, bounded — N/A (no realloc; fixed `InitSpace`)
- [x] `space` correct; all `Vec`/`String` bounded — `#[max_len(...)]` on every Vec/String (`state.rs:57-198`, `constants.rs:11-19`)
- [ ] No trusted value taken as unchecked instruction argument — settlement outcome & parlay leg results are trusted council inputs (by design, **F-INFO-1**); leg results have no Pending→final guard, enabling churn/grief (**F-05, LOW** / **F-03, MEDIUM, Code**) — `record_leg_result.rs:54`

## Economic Invariants
- [x] All invariants named and each has an enforcing line (design-spec §6) — 6 invariants enumerated
- [x] `overflow-checks = true` (release) + `checked_*` on value math — `Cargo.toml:8`; `checked_mul/div/sub` + `u128` intermediates (`util.rs:6-21`)
- [x] Rounding always favors the protocol, consistently — payout floors (`util.rs:6-13`); `collateral > 0` rejects zero-risk quotes (`submit_quote.rs:59`, `accept_quote.rs:82`)
- [-] First-depositor / share-inflation mitigated — N/A (no shares/pool; per-position escrow, not a vault)
- [-] Oracle staleness + confidence checked; no flash-manipulable spot price — N/A (no oracle; council settles)
- [-] Slippage bounds (`min_out`/`max_in`) enforced — N/A (no swaps); odds floor via `min_odds_bps` is the analogous user bound (`post_rfq.rs:43`, `submit_quote.rs:52`)
- [-] Fees bounded (≤ cap) and inside the conservation invariant — N/A (zero house edge; only the fixed 0.01 SOL spam deposit, fully refunded)

## Upgrade & Governance
- [ ] Upgrade authority is multisig (not single hot key) or immutable-by-plan — **disposition undocumented** (**F-02, HIGH, Launch**); set Squads v4 multisig+timelock or `--final` before mainnet
- [ ] Immutable-vs-upgradeable decision is explicit — not decided; README only defers mainnet "behind a security audit"
- [x] Admin blast radius enumerated and bounded — bounded to *nothing*: `config.authority` is set but never read; no admin instruction can move funds/change council/pause (`initialize.rs:43`)
- [~] Multisig + timelock on highest-impact actions — settlement uses native in-program threshold voting (no single-admin settle, `execute_settlement.rs:75-78`); the **upgrade** action is not yet multisig-gated (F-02)
- [ ] Emergency pause exists and never blocks user exit — **no pause**; posture is fail-closed (funds stay escrowed). Acceptable for MVP; revisit before mainnet

## Compute, Limits & DoS
- [x] No unbounded iteration; loops fit compute budget at max scale — loops bounded: council ≤ 9 (`is_council`, voters), parlay legs ≤ 4 (`execute_settlement.rs:97-111`); `MAX_*` constants in `constants.rs`
- [x] No account/collection grows without a close/cleanup path — RFQs close on cancel/expire/settle; positions close on settle; permissionless `expire_rfq` crank cleans abandoned RFQs
- [x] Tx size & account count within limits (or v0 + ALTs by design) — modest account counts per ix; no large remaining-accounts arrays
- [x] Heavy instructions set compute limit; peak CU measured — no heavy ix; bounded small loops. (Recommend measuring `execute_settlement` peak CU on devnet to confirm.)
- [x] Caller (not protocol PDA) funds account rent; no squattable slots — payers are the user/cranker on every `init`/`init_if_needed` (`post_rfq.rs:28`, `execute_settlement.rs:50,60`)
- [ ] **(open) Settlement liveness** — a single council member can flip a parlay leg post-vote and revert `execute_settlement`, trapping funds (**F-03, MEDIUM, Code**) — `record_leg_result.rs:54` vs `execute_settlement.rs:96-112`
